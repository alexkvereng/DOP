Утилита logman для управления журналом событий
Имеется текстовая база данных - “журнал событий” – следующей структуры:
<номер-по-порядку> <логический-номер> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
<строка-с-описанием-события> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
<номер-по-порядку> <логический-номер> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
<строка-с-описанием-события> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
<номер-по-порядку> <логический-номер> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
<строка-с-описанием-события> <СИМВОЛ ПЕРЕВОДА СТРОКИ>
и т.д.

Например:
1 2 \n
Событие номер два \n
2 1 \n
Событие номер один \n
3 3 \n
Событие номер три \n

Для управления этой базой требуется разработать утилиту logman.exe со следующим интерфейсом.
1) вызов logman.exe для записи в журнал событий:
logman -w <логический-номер> <расположение-файла-журнала-событий>
из файла “data.in” в домашнем каталоге  logman берет строку <строка-с-описанием-события>. Перед записью в файл символы перевода строки заменяются на пробелы.
В стандартный поток вывода пишется <номер-по-порядку> (на 1 больше последнего записанного, начиная с 1) для вновь созданной записи или 0 в случае ошибки. Если для заданного логического номера запись уже имеется, то если <строка-с-описанием-события> у этой записи и в файле “data.in” совпадают, то возвращается номер по порядку этой записи, иначе — 0.
2) вызов logman.exe для чтения из журнала событий:
logman -r <номер-по-порядку > <расположение-файла-журнала-событий>
создается в файл “data.out” в домашнем каталоге программы в который logman пишет строку <строка-с-описанием-события> для переданного на входе значения <номер-по-порядку >. В стандартный поток вывода возвращается логический номер этой записи или 0 в случае ошибки.
Для работы с файлами и строками в logman.exe использовать средства стандартной библиотеки, реализация – на языке С++ или С.
